웹 기초

1. 요청과 응답 이해하기
 
  1) 요청 전문

    헤더(header)
      - 요청하는 쪽의 정보
      - 요청 방식(Method)
      - 요청 URL
      - 요청 IP
      - 브라우저 정보, 언어...
        User-Agent : 브라우저 종류

    바디(body)
      - 서버에 전송하는 데이터

  2) 응답 전문

    헤더(header)
      - 응답 바디(body)의 형식에 대한 정보
        - Content-type

      - 서버쪽에서 브라우저쪽에 명령
        - Cache-control
        - Refresh :
        - Set-Cookie : 쿠키 설정
        - Location : 주소 -> 주소 이동

    바디(body)
      - 서버가 응답한 데이터

2. HTTP( HyperText transfer Protocol )

3. 헤더(header)

4. HTTP 상태 코드
  응답 헤더 : status

  2xx : 성공 응답
    200 : 정상 응답
    201 : CREATED - 작성됨
    202 : ACCEPTED
    
  3xx : 리다이렉션 메시지
    301, 302 -> 페이지 이동
    304 : 캐시됨

  4xx : 클라이언트 에러 응답(요청쪽)
    400 : Bad Request - 잘못된 요청
    401 : Unauthorized - 접속 권한 없음
    403 : Forbidden - 권한이 없는 경우, 현재 서버 디렉토리가 접근 권한이 없는 경우
    404 : Not Found - 페이지 없음
    405 : Method Not Allowed - 요청 방식이 서버에 정의가 안되어 있는 방식으로 접근 한 경우,
    
  5xx : 서버 에러 응답
    500 : Internal Server Error - 프로그램 소스 상의 오류, 서버 환경에 문제 ...
    501, 502 : Service Not Available -
  

5. HTTP 요청 메서드
  GET - 조회 : 서버에서 자원을 조회
    https://n.news.naver.com/mnews/article/008/0004906101
    ?sid=105

    ? : 질의(query)
      - 쿼리스트링
        ? 이름 = 값 & 이름 = 값 ...

          https://news.naver.com/main/main.naver
          ?
          mode=LSD
          &
          mid=shm
          &
          sid1=105

          참고) GET 형식
            DELETE : GET 방식의 일종


  POST - 작성 : 서버에 기록을 위한 용도
    - 요청 헤더
      - 요청 바디 데이터의 종류 : 서버가 종류를 알아서 서버에서 변환해서 사용 가능
      - Content-Type : 
        - application/x-www-form-urlencoded : 일반적인 양식 형식 키 = 값 & 키 = 값..

        - application/json : JSON의 바디 데이터 
                            참고) {키 : 값, 키 : 값...}


    - 요청 데이터 : 요청 바디
      subject=%EC%A0%9C%...
      &
      content=%EB%82%B4%...

      - 한글은 전송 X -> URL 인코딩 - 16진수

    참고) POST 형식

      PUT : 데이터 치환

      PATCH : 부분 데이터 수정
